# User Story List — Venue Vendors (Assignment 2)

**Format:** User Story = *"As a (persona) I want to (task) so that (goal)"* · Acceptance Criteria = *"Given … When … Then …"*

This markdown file is same thing as UserStoryList.xlsx

| Epic | Persona | User Story | Acceptance Criteria |
| --- | --- | --- | --- |
| Auth | Visitor | As a visitor I want to sign up with my email, password, name, phone and role so that I can get an account on Venue Vendors | Given I am on the sign-up page, When I submit a unique email, a password with at least 6 characters including 1 uppercase 1 lowercase and 1 special character, my name, phone and role, Then my account is created and I am taken to the sign-in page |
| Auth | Visitor | As a visitor I want weak passwords to be rejected so that my account stays secure | Given I am on the sign-up page, When I enter the password "abc", Then the form (and the API) reject it with a clear password error and no account is created |
| Auth | Visitor | As a visitor I want duplicate emails to be rejected so that one email maps to one account | Given an account with my email already exists, When I try to sign up with the same email, Then I see an "Email already taken" error and no second account is created |
| Auth | Registered User | As a registered user I want to sign in with my email and password so that I can access my dashboard | Given I have an account, When I enter the correct email and password, Then I am signed in, I see a welcome confirmation and I am redirected to my role's dashboard |
| Auth | Registered User | As a registered user I want wrong credentials to be refused so that nobody can access my account | Given I have an account, When I enter an incorrect password, Then I see "Invalid email or password" and I am not signed in |
| Auth | Registered User | As a registered user I want a JWT stored after sign-in so that protected pages and API calls work automatically | Given I have signed in successfully, When the app makes an API request, Then the request includes my Authorization Bearer token and the protected endpoint returns my data |
| Auth | Registered User | As a registered user I want to log out so that my session cannot be reused on a shared computer | Given I am signed in, When I click Logout in the navbar, Then my token and user are cleared and I am returned to the sign-in page |
| Auth | Registered User | As a registered user I want to view my profile so that I can confirm my stored details | Given I am signed in, When I open my profile page, Then I see my name, email, phone and date of joining |
| Hirer Browsing | Hirer | As a hirer I want to browse all available venues so that I can find one for my event | Given I am a signed-in hirer, When I open Browse Venues, Then I see at least 3 venues each showing name, capacity, location and a View button |
| Hirer Browsing | Hirer | As a hirer I want to search venues by name or location so that I can find a venue quickly | Given the venue list, When I type a name or location into the search box, Then only venues matching that text are shown |
| Hirer Browsing | Hirer | As a hirer I want to filter venues by capacity so that I only see venues big enough for my guests | Given the venue list, When I set a minimum capacity, Then only venues with capacity greater than or equal to that number are shown |
| Hirer Browsing | Hirer | As a hirer I want to see a single venue's full details and available timeslots so that I can decide whether to apply | Given a venue card, When I click View, Then I see the venue's description, amenities, price and the list of available (non-blocked) dates |
| Hirer Browsing | Hirer | As a hirer I want recommended suitability chips so that I can see how well a venue fits my event type | Given I picked an event type from the "I'm planning a..." dropdown, When I view a venue, Then I see coloured chips of the matched keywords and a suitability match score |
| Hirer Browsing | Hirer | As a hirer I want to see Featured venues highlighted so that I can quickly spot venues the admin has promoted | Given some venues have been marked featured by an admin, When I open Browse Venues, Then those venues appear in a Featured section |
| Hirer Booking | Hirer | As a hirer I want to apply to hire a venue so that I can request it for my event | Given a venue page, When I submit valid event details (event name 2-100 chars, guests >= 1 and <= venue capacity, date today or later), Then a booking is created with status pending and I see a success message |
| Hirer Booking | Hirer | As a hirer I want invalid event details to be blocked so that I don't submit a broken application | Given the apply form, When I enter 0 guests or a past date or an event name containing < or >, Then the form shows an inline error and nothing is submitted |
| Hirer Booking | Hirer | As a hirer I want to be stopped from booking a blocked timeslot so that I don't request a date the vendor closed | Given a venue timeslot has been blocked by the vendor, When I submit an application for that timeslot, Then the API responds 409 Conflict with "timeslot blocked" and no booking is created |
| Hirer Booking | Hirer | As a hirer I want to see my booking history with statuses so that I can track my applications | Given I have submitted applications, When I open Booking History, Then I see each booking with its venue, event details and current status (pending/approved/rejected) |
| Hirer Booking | Hirer | As a hirer I want to save venues to a ranked list so that I can compare my favourites later | Given a venue, When I click Save, Then it is added to my Saved Venues list with a rank |
| Hirer Booking | Hirer | As a hirer I want to reorder my saved venues so that my top choice is first | Given I have saved venues, When I move a venue up or down, Then its rank updates and the list re-orders |
| Hirer Booking | Hirer | As a hirer I want to remove a saved venue so that my list stays relevant | Given a saved venue, When I click Remove, Then it is deleted from my saved list |
| Hirer Compliance | Hirer | As a hirer I want to upload compliance documents so that vendors can trust me | Given the Compliance page, When I upload a PDF of 5 MB or less, Then the document is stored and listed with its type and upload date |
| Hirer Compliance | Hirer | As a hirer I want non-PDF or oversized files rejected so that only valid documents are stored | Given the Compliance page, When I choose a non-PDF file or a file over 5 MB, Then the form rejects it with an inline error and nothing is uploaded |
| Hirer Compliance | Hirer | As a hirer representing a business I want to provide my ABN so that my business documents are complete | Given I tick "I'm representing a business", When I submit, Then the ABN field is required and my business certificate is recorded |
| Hirer Compliance | Hirer | As a hirer I want to see my compliance score so that I know how trustworthy my profile looks | Given I have uploaded documents, When I open the Compliance page, Then I see a score based on how many of the required documents I have provided |
| Hirer Reputation | Hirer | As a hirer I want to see my reputation history so that I know how vendors have rated me | Given I have past completed bookings, When I open the Reputation page, Then I see an average star rating shown in 0.0 to 5.0 format and a history of past venues |
| Hirer Reputation | Hirer | As a hirer with no reviews I want a sensible empty state so that the page does not look broken | Given I have no completed bookings, When I open the Reputation page, Then the average is shown as not yet rated rather than an error |
| Hirer Account | Hirer | As a hirer I want to update my own details so that my contact information stays current | Given I am a signed-in hirer on My Details, When I change my phone or name and save, Then the change is persisted to the database and shown on reload |
| Vendor Venue Mgmt | Vendor | As a vendor I want to see only my own venues so that I manage just my listings | Given I am vendor A, When I open My Venues, Then I see only venues where the vendor is A and never another vendor's venues |
| Vendor Venue Mgmt | Vendor | As a vendor I want to create a venue so that hirers can find and book it | Given the Add Venue form, When I submit a name (2-100 chars), capacity (1-10000) and location, Then a new venue owned by me is created |
| Vendor Venue Mgmt | Vendor | As a vendor I want to edit and delete my venues so that my listings stay accurate | Given one of my venues, When I update or delete it, Then the change is saved; and when I try to edit a venue I do not own the API responds 403 Forbidden |
| Vendor Venue Mgmt | Vendor | As a vendor I want to block and unblock timeslots so that I can close dates I am unavailable | Given one of my venue's free timeslots, When I block it, Then it is no longer bookable; and when I block a timeslot that has an approved booking the API responds 409 with "timeslot has active booking" |
| Vendor Bookings | Vendor | As a vendor I want to see all booking requests for my venues with hirer reputation and compliance so that I can judge applicants | Given hirers have applied to my venues, When I open Applications, Then I see each request with the hirer's name, reputation stars and compliance score |
| Vendor Bookings | Vendor | As a vendor I want to approve or reject a booking so that I control who uses my venue | Given a pending booking, When I click Approve, Then its status becomes approved and the timeslot is no longer available to others; When I click Reject, Then its status becomes rejected |
| Vendor Bookings | Vendor | As a vendor I want to leave a credibility comment on a hirer so that other vendors have context | Given a completed booking, When I add a comment with a credibility tag, Then it is saved against that hirer |
| Vendor Analytics | Vendor | As a vendor I want charts of my bookings over a chosen time range so that I can understand demand | Given the Infographic Report page, When I pick This week / This month / Last month / All time, Then at least 3 charts (bookings per venue, status breakdown, applicants never approved) refresh for that range |
| Vendor Analytics | Vendor | As a vendor with no bookings I want the stats page to still load so that a new account is not broken | Given a brand-new vendor with no bookings, When I request stats, Then the API returns 200 with empty arrays rather than a 500 error |
| Vendor Account | Vendor | As a vendor I want to update my own details so that hirers see correct contact information | Given I am a signed-in vendor on My Details, When I change and save my details, Then the change is persisted to the database |
| Admin Mgmt | Admin | As an admin I want to log in with admin credentials so that only staff can access the admin dashboard | Given the admin login page, When I submit the admin username and password, Then I receive an admin token and reach the admin dashboard; the admin app is completely separate from the public site |
| Admin Mgmt | Admin | As an admin I want to create, update and delete any venue so that I can keep the catalogue clean | Given the admin venues table, When I create, edit or delete a venue, Then the change is saved in the shared database |
| Admin Mgmt | Admin | As an admin I want to assign a vendor to a venue so that ownership can be corrected | Given a venue and a vendor, When I assign that vendor to the venue, Then the venue's vendor is updated |
| Admin Mgmt | Admin | As an admin I want to feature a venue so that it is promoted on the public site | Given a venue, When I toggle Feature on, Then venue.isFeatured becomes true and the venue appears in the public Featured section |
| Admin Reports | Admin | As an admin I want a report of the top 3 venues with their most popular day and timeslot so that I can advise vendors | Given approved bookings exist, When I open Reports, Then I see the top 3 venues each with their most popular weekday and timeslot |
| Admin Reports | Admin | As an admin I want a report of the top 3 most active applicants so that I can recognise engaged hirers | Given hirers with at least 3 applications, When I open Reports, Then I see the top 3 applicants ranked by approved bookings divided by total applications |
| Admin Notifications | Admin | As an admin I want to see applicants who have never been approved so that I can follow up with them | Given some hirers have zero approvals across all their applications, When I open the relevant report, Then those applicants are listed separately |
| Quality | Developer | As a developer I want 6 contextual unit tests in the backend so that key risks are protected and the HD criteria are met | Given the backend test suite, When I run npm test in /backend, Then all 6 tests (3 hirer, 3 vendor) pass and each has a comment block explaining its context |
| Quality | Developer | As a developer I want all 4 apps to build cleanly so that the submission compiles for full marks | Given the four sub-projects, When I run npm run build in each, Then TypeScript compiles with zero errors |
| Deployment | Developer | As a developer I want all 4 apps deployed on Render so that the markers can access live URLs | Given the four apps, When I open each Render URL in a fresh browser, Then the app loads without console errors and the 4 URLs are listed in the root README |
